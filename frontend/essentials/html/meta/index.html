Meta标签属性

前端开发笔记
字数统计: 1.7k字   |   阅读时长≈ 7分钟
2021-07-22

<meta> 标签是 HTML 里的一个辅助性标签，通常用于定义页面编码语言、搜索引擎优化、自动刷新并指向新的页面、控制页面缓冲、响应式视窗等方面。

Tips： name 属性 和 http-equiv 属性不能同时设置。

name

name 属性主要用于描述网页，对应属性是 content。

  • keywords

    为搜索引擎提供的关键字列表，content中的内容需要用英文半角逗号隔开。

  • description

    告诉搜索引擎你的网站主要内容。

  • robots

    robots用来告诉搜索机器人哪些页面需要索引，哪些页面不需要索引。

    • all (默认)

    • none

    • index

    • noindex

    • follow

    • nofollow

  • googlebot

    定义谷歌浏览器的抓取行为。

  • google

    • notranslate
不提供此页面的翻译。

  • nositelinkssearchbox

    不显示网站链接的搜索框。

  • author

    标注网页的作者。

  • generator

    用于标明网页是什么软件做的。(如使用 WordPress、Dreamweaver 等软件生成的网页。)

  • copyright

    标注所属版权。

  • subject

    关于网站主题的简短描述

  • rating

    基于网站内容给出年龄分级

  • revisit-after

    设定搜索引擎每隔多少天访问一次页面。

  • viewport

    • width

      取值为：整数或device-width，定义viewport的像素宽度，或允许viewport适应设备的屏幕宽度。

    • height

      取值为：整数或device-height，定义viewport的高度。

    • initial-scale

      取值为：0.0 - 10.0，定义设备宽度（纵向模式下的设备宽度或横向模式下的设备高度）与viewport大小之间的比例。

    • maximum-scale

      取值为：0.0 - 10.0，定义最大的缩放级别。它必须大于或等于minimum-scale，否则视为未定义。浏览器设置可以忽略此规则，iOS10+ 默认情况下忽略它。

    • minimum-scale

      取值为：0.0 - 10.0，定义最小的缩放级别。它必须小于或等于maximum-scale，否则视为未定义。浏览器设置可以忽略此规则，iOS10+ 默认情况下忽略它。

    • user-scalable

      取值为：yes 或 no，如果设置为no，用户将无法放大网页。默认值为yes。浏览器设置可以忽略此规则，iOS10+ 默认情况下忽略它。

  • format-detection

    移动端常用于，关闭 → 点击数字作为电话号码跳转到拨号界面

  • renderer

    用于指定双核浏览器默认以何种方式渲染页面。如 360浏览器、QQ浏览器等双核浏览器。

    • webkit

      默认webkit内核。

    • ie-comp

      默认IE兼容模式。

    • ie-stand

      默认IE标准模式。

  • application-name

    定义在网页中运行的应用程序的名称，通常在 webapp 下才使用。

  • apple-mobile-web-app-capable

    是否显示默认的苹果工具栏和菜单栏

  • apple-mobile-web-app-status-bar-style

    状态栏的背景颜色（最顶部的系统时间/电量/数据流量/WiFi 那一栏）

    • default 白色

    • black 黑色

    • black-translucent 灰色半透明

  • apple-touch-fullscreen

    添加到主屏幕后，是否全屏显示

  • msapplication-tap-highlight

http-equiv

http-equiv类似于HTTP的头部协议，它回应给浏览器一些有用的信息，以帮助正确和精确地显示网页内容。对应属性是 content。

  • expires

    指定网页在缓存中的过期时间，一旦网页过期，就会发出请求从服务器获取数据。

    • GMT 时间格式 例：Thu, 22 Jul 2021 06:35:31 GMT， 可使用 new Date().toGMTString() 生成。

    • 0 或 -1， 不缓存

  • pragma

    用于设定禁止浏览器从本地机的缓存中调阅页面内容，设定后一旦离开网页就无法从Cache中再调出。

    Tips: 设置后，无法查看离线缓存。

  • refresh

    自动刷新并指向新页面。

    • second 几秒后执行，如下示例 5，意为 5 秒后刷新。

    • url 刷新后的新页面地址

  • X-UA-Compatible

    指定浏览器以某种版本来渲染页面。

  • x-dns-prefetch-control

    关闭 DNS 预加载。

  • cache-control

    指定请求和响应遵循的缓存机制。

    • no-cache

      先发送请求，与服务器确认该资源是否被更改，如果未被更改，则使用缓存。

    • no-store

      不允许缓存，每次都要去服务器上，下载完整的响应。（安全措施）

    • public

      缓存所有响应，但并非必须。因为max-age也可以做到相同效果

    • private

      只为单个用户缓存，因此不允许任何中继进行缓存。（比如说CDN就不允许缓存private的响应）

    • max-age

      表示当前请求开始，该响应在多久内能被缓存和重用，而不去服务器重新请求。例如：max-age=60 表示响应可以再缓存和重用 60 秒。

    • no-siteapp

      禁止百度转码

  • content-security-policy

    定义当前页面的内容策略。内容策略主要指定允许的服务器地址和脚本端点，这有助于防止 跨站脚本攻击 （Cross-site scripting, XSS）攻击。

    Tips: 这行代码需要放在比较靠前的位置，比如 style 和 script 等外部文件前。

charset

用于设定页面字符集的语言编码。

  • GB2312 简体中文

  • BIG5 繁体中文

  • iso-2022-jp 日文

  • ks_c_5601 韩文

  • ISO-8859-1 英文

  • UTF-8 世界通用
